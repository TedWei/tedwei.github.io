使用 node & gulp 打造自己的工作流

2016-05-20

现在gulp和Node是我工作中不可缺少的一部分，我依然还只是做着传统的前端工作。

我使用gulp来压缩 autoprefixer scss babel inline file-include 等之前已经在用的功能。

现在加上node。我开始写一些文件替换工作。
比如我使用zepto作为基础组件,但是不太想使用使用整个包。那我在本地的时候会使用require 动态加载需要的组件。因为依赖相同。那么我就可以动态的转换成requirejs需要的格式来进行加载。

在发布的时候 我又需要将这些动态加载的部分组件进行合并。那我又可以进行合并后 在转换成requirejs需要的格式

我们使用jenkins进行应用发布。我构建了一个简易的build.js 填入你的jenkins地址，job的名字 你的帐号 密码。然后node build.js 就可以发布了

在本地调试或者mock的时候 我需要将url指向本机地址或者mock，但是上线又不太需要。 我将这部分提取出来，动态的根据我的gulp task进行文件替换。

我们使用svn来进行代码管理。我就每次将gulp prod后的文件 拷贝到对应的本机svn地址。然后执行 svn_autocommit.sh 。会自动添加，删除 进行commit。在根据commit的输出判断是否进行jenkins的build。

这样每次开发应用。我只需要 gulp dev 开启本地的实时预览开发环境。
gulp prod && gulp prod:build 进行代码编译，发布，应用构建。

还有很多可以自动化的工作：自动化测试，UI测试，生产环境的本机代理调试，自动的mock数据生成，CDN等。
